# FAQ

## Is GRITS just a hardening framework?

No. Hardening is one foundational layer inside GRITS. The bigger goal is lifecycle governance and runtime assurance for AI agents and LLM systems.

## Is GRITS only for enterprise teams?

No. GRITS is deliberately dual-track. It gives immediate adoption artifacts for operators and self-hosters, and reference artifacts for enterprise-scale governance.

## Is GRITS a product?

No. GRITS is a framework package. It can inform products and control planes, but it is not itself a runtime or managed platform.

## Does GRITS replace NIST AI RMF, OWASP, ATF, or AAGATE?

Not necessarily. In many cases GRITS should sit underneath or alongside them as the implementation and operationalization layer.

## Why are scripts versioned but docs are not?

The repository itself provides version control for docs. Script versioning is kept because tools often change behavior in ways that users may want to track explicitly.
