# Terminology and domain model

## Core terms

| Term | Meaning |
|---|---|
| Agent | a governed software object that can use tools, access data, invoke workflows, or act with partial or full autonomy |
| LLM App | a governed LLM-based system such as a chatbot, copilot, prompt wrapper, or RAG app that may not meet the threshold for agent autonomy |
| Managed object | any governed system tracked under GRITS, including Agents and LLM Apps |
| Lifecycle state | the current governance state of a managed object |
| Trust status | the current confidence or governance standing assigned to a managed object |
| Runtime signal | an event or evidence point relevant to continuous governance |
| Baseline | a minimum practical control set for a deployment pattern |
| Playbook | an action-oriented sequence for deployment, remediation, or operations |
| Scorecard | a summarized governance and assurance output built from applicable requirements and evidence |
